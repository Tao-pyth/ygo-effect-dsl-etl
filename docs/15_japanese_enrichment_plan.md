# 15_japanese_enrichment_plan — 日本語名・日本語効果文の後追い取得（設計メモ）

## 目的
初期実装では YGOPRODeck の英語中心データを主として取り込みつつ、
将来「別口」で日本語名・日本語効果文を取得して埋められるように、最初から器を用意する。

## 方針（ETLの責務）
- DB と JSONL に以下の列/キーを **必ず持つ**（初期は空文字でOK）
  - `name_ja`
  - `card_text_ja`
  - `card_info_ja`（任意：日本語側の補足情報がある場合）
- 日本語取得の実装は “別コマンド” として追加できる設計にする（例：`sync-ja` / `enrich-ja`）
- 日本語が取得できなくても、既存の export 契約は壊さない

## 具体案（おすすめ）
- `enrich-ja` コマンド（将来追加）
  - 入力：SQLite（cidで結合）
  - 出力：SQLiteの `name_ja` / `card_text_ja` を更新
  - 再実行可能（冪等）
- `export` は常に “現在DBに入っている値” を吐く（後追い結果が反映される）

## 注意点
- 日本語ソースのライセンス/利用規約に配慮する（公開時の同梱範囲を慎重に）
